2. 各レイヤーの責務

coins.js — コイン残高 (localStorage `tamsic_wallet` + Cognito `custom:coins` 同期)、`window.TAMSICCoins`

3. レター送信フロー (主要ユースケース)

  1. ユーザーがアーティストページで「30 coin で1回フル試聴」ボタン押下
  2. coins.js spendCoins() → auth.js spendCoinsOnCognito() で custom:coins 減算 + localStorage 更新
  3. unlockFullTrack(track) で audio 再生開始 + lyrics-inner-{trackId} 内に TAMSICLetter.render() 呼出
  4. letter-frames.js pickFrame() で 18枠から抽選 (誕生日/周年/レア/通常の優先順位)
  5. letter-content.js pickClosing() で closing 一文抽選
  6. DOM 組立 → lyrics-guard.protectLetter() で 5層保護適用
  7. 便箋表示 + 「別の一文」「この手紙をメールで受け取る」ボタン
  8. 受信ボタン押下 → TAMSICAuth.hasReceivedLetter() で重複チェック
  9. 確認モーダル (Dear ●● 入力 + 送信先メアド表示)
  10. 送信ボタン → nickname を Cognito に保存 → POST /tamsic-send-letter.workers.dev/ (Bearer access_token)
  11. Worker 側で /oauth2/userInfo で本人確認 + 抽選を再実行 (クライアント信用しない)
  12. HTML メール構築 (インライン CSS、@fontsource @import) → Resend API
  13. Resend → SMTP → ユーザー受信箱
  14. Worker から フロントへ { ok, frame, closingText, sentDate, recipientHash } 返却
  15. TAMSICAuth.appendLetterHistory() で custom:letterHistory に1件追加 (FIFO 上限10件)
  16. 完了モーダル表示

4. 18枠の優先順位

順位判定結果枠備考
1誕生日 (今日 = custom:birthday の月日)O強制、紙吹雪 + Banner
2周年 (今日 = custom:registeredAt の月日)P強制、星 + N YEAR Banner
3レア Q 抽選 (1%)Q当選で確定、★ RARE ★ バッジ
4レア R 抽選 (Q未当選後の 5%)R結果出現率 ≈ 4.95%、☆ UNCOMMON ☆
5通常抽選 (artist プール ∪ 季節枠)A〜J + 期間中の K〜N直前と違う枠を優先

AttributeTypeMax LengthFormat用途

8. 既知の制約

  • letterHistory 2048字上限 — FIFO 10件運用、超過時は古い順に削除。長期 DynamoDB 化検討
  • Resend 月3,000通無料枠 — 超過時は有料プラン or AWS SES 移行
  • Outlook 互換性未テスト — 受信側報告ベースで微調整
  • 音源 public access — assets/audio/ が直 URL 推測可能、R2 + signed URL 化が将来課題
  • Square Webhook 未実装 — コイン付与は手動「購入を反映する」ボタン
  • creatorNote / closings 未投入 — 12曲中 1曲のみ確定、残り 11曲 placeholder
